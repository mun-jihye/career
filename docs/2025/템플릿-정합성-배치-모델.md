# 템플릿 정합성 배치 모델 개선 및 적용

> **dinoV2 → dreamsim** 모델 전환으로 **육안 검사 대상 88% 감소**를 이끈 정합성 배치 모델 개선·적용 내역.  
> **출처**: 2025 성과아카이브, [AI-785](https://miridih.atlassian.net/browse/AI-785)

---

## 1. 빌드업·결과 요약

| 구분 | AS-IS (dinoV2) | TO-BE (dreamsim) |
|------|----------------|------------------|
| **템플릿 수** | 69,979개 | 69,494개 |
| **“유사하다” 판단** | 10,940개 | 62,269개 (약 **5.7배** 증가) |
| **육안 검사 대상** | **59,039개** | **7,225개** → **약 88% 감소** |

- **목표**: “유사하다”고 판단된 템플릿은 육안 검사에서 제외하고, 나머지만 검사하여 **운영 부담 감소**.

---

## 2. 내가 한 작업 (역할)

- **정량 비교 설계**: **grid size**를 1부터 25까지 다양하게 설정하고, **기존 모델(dinoV2)·개선 모델(dreamsim)** 두 종류를 적용해 **총 50가지 버전**으로 유사도 결과를 추출·비교.
- **지표 분석**: **TP, TN, FP** 등 주요 지표를 **수치로 확인**하고, “어떤 모델이 더 좋아 보인다”가 아니라 **데이터 기반으로 개선 방향** 도출.
- **의사결정 지원**: 결과를 리드(장주영)와 공유해 **객관적 기준**으로 dreamsim 적용·배치 반영 결정.

---

## 3. 사용한 툴·기술

| 구분 | 툴·기술 |
|------|----------|
| **모델** | dinoV2 (기존), dreamsim (개선) |
| **정합성·유사도** | 이미지 유사도 점수, “유사/비유사” 판단 기준 |
| **실험** | grid size 1~25 × 2모델 = 50가지 버전, 유사도 결과 추출·비교 |
| **지표** | TP, TN, FP 등 (정확도·재현율 등 정량 지표) |
| **참고** | Jira [AI-785](https://miridih.atlassian.net/browse/AI-785) |

## 4. 워크플로우 (개념)

1. **입력**: 템플릿 썸네일(또는 대표 이미지) 쌍 또는 목록.
2. **유사도 계산**: grid size별로 영역을 나누어, 각 영역(또는 전체)에 대해 dinoV2 / dreamsim으로 유사도 점수 계산.
3. **판단**: 임계값 또는 규칙에 따라 “유사하다”/“유사하지 않다” 분류.
4. **집계**: 50가지 버전별로 “유사” 건수·육안 검사 대상 건수 집계.
5. **지표 계산**: TP, TN, FP 등으로 모델별 성능 비교.
6. **배치 적용**: dreamsim 채택 시, 정합성 배치 프로세스에서 dreamsim 모델을 사용하도록 반영.

(상세 배치 스텝·스크립트 경로는 팀 내부 문서·AI-785 하위 태스크 참고.)

---

## 5. 핵심가치 연결 (성과아카이브 기준)

- **데이터 기반**: 주관이 아닌 **숫자와 사실**로 모델 성능을 비교하고, 개선 방향을 도출. 50가지 버전·TP/TN/FP 분석으로 의사결정 근거 마련.
- **임팩트**: 육안 검사 대상 **88% 감소**로 운영 부담을 크게 줄임.
