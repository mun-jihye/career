## 1. 문제 정의

- 디자인허브에 제출된 요소 중 **일부 저품질 요소가 심사 과정에서 필터링되지 않고 등록**되는 문제 해결.
- 관련 문서: [디자인허브 725221588](https://miridih.atlassian.net/wiki/spaces/R4XkdUZCxgHI/pages/725221588).

---

## 2. 필요한 데이터 (공통)

- **필요 컬럼**: action_item(요소 id), action_item_type, user_pick_cnt, **이미지 url**.

## 3. 데이터셋 정의

### 3.1. 디자인허브 측면 데이터셋

| 구분 | 내용 |
|------|------|
| **승인** | 유지. |
| **거부** | 유지 + **승인되었다가 신고로 거부된 케이스** (신고로 거부된 케이스 CSV는 Slack 등으로 공유). |
| **라벨링 기준** | [요소 거부 라벨링 기준](https://miridih.atlassian.net/wiki/spaces/R4XkdUZCxgHI/pages/722829784). |
| **데이터브릭스** | [Notebook 경로](https://miridih-workspace.cloud.databricks.com/editor/notebooks/784525801361233?o=2263682544665348). |

**디자인허브 승인/거부 쿼리 요약**

- **승인**: element_item, content_review_item, element_thumbnail_item 조인, create_date ≥ 2024-03-01, active_status='HIDDEN', cri.status='APPROVAL', eti.size='ORIGINAL', admin_memo 제외 조건, RANDOM() LIMIT 700000.
- **거부**: cri.status='REJECTION', reason_korean_common, MIN(store_file_url) 등.

### 3.2. 검탐색 측면 데이터셋 (진행 여부는 팀 정책 따름)

| 구분 | 내용 |
|------|------|
| **고품질** | 최근 6개월, **유저픽이 높은 요소 70만 건**. |
| **저품질** | **노출 대비 채택률이 낮은 요소 50만 건**. |
| **조건** | '검색' 한정 (뷰, 비슷한템플릿찾기, 더보기, 기타 제외). |

**데이터브릭스**: [검탐 요소 품질 분류 데이터셋 Notebook](https://miridih-workspace.cloud.databricks.com/editor/notebooks/518103658116735?o=2263682544665348).

- 고품질: events에서 search_status='검색', event_name='action_element' 등으로 user_pick_cnt 집계, 상위 700000.
- 저품질: view_element / action_element 조인, ratio(채택/노출) 낮은 순 400000.

## 4. 파이프라인 AS-IS / TO-BE

### 4.1. AS-IS

1. 리서처가 **MA 스쿼드에 온콜로 데이터 요청**.
2. MA가 **데이터브릭스**로 요청 데이터 수집 후 **데이터브릭스 링크** 전달.
3. 리서처가 **CSV 추출** 후 **GPU 서버에 업로드**.
4. **csv 파싱·이미지 적재 스크립트** 실행:
   - CSV에서 이미지 경로 추출 → **file.miricanvas.com**으로 이미지 요청 및 적재.

### 4.2. TO-BE

- Confluence 문서 상 TO-BE 본문은 비어 있음.  
- 팀 목표는 **Airflow 등으로 데이터 적재 요청 → 이미지 처리 → GPU 연산**까지 자동화 (동일 스페이스 “딱 맞는 디자인 찾기” 파이프라인 TO-BE와 같은 방향).

## 5. 내가 한 작업 (역할)

- **데이터셋 정의**: 디자인허브(승인/거부)·검탐(고품질/저품질) 데이터셋 정의, 필요 컬럼·조건 정리.
- **데이터브릭스 쿼리**: 디자인허브·검탐용 SQL 쿼리 작성·문서화, Notebook 경로 정리.
- **파이프라인 문서화**: AS-IS 4단계(리서처→MA→데이터브릭스→리서처 csv·GPU→스크립트) 정리, TO-BE 방향 연결.
