### 4.2. xml-image-csv-converter 자동화 (Airflow, AWS Batch) (핵심 성과 2)

| 구분 | 내용 |
|------|------|
| **AS-IS** | 대량 XML→CSV 추출 시 **EC2 수동 기동·종료**, 최대 6대(r7i.xlarge) 병렬. 작업 단위 스케줄링·실패 재시도·리소스 회수 미자동화, 운영 개입·인프라 부담 |
| **목표** | 일배치(Job) 단위 실행 가능한 **컨테이너화**, EC2 직접 운영 없이 **자동 확장·자동 종료** 구조 |
| **기여도 / 내 역할** | • **CI/CD**: 코드 변경 시 Docker 이미지 빌드 → ECR 업로드 자동화 <br>• **Airflow DAG** 기반 배치 실행: XML→image·csv 변환 작업 실행 시점·재시도·순차/병렬 제어 <br>• **AWS Batch** 도입: EC2 직접 기동 → 컨테이너 기반 Job 실행, 작업 수 증가 시 자동 확장, 종료 후 리소스 자동 해제 |
| **결과** | 대량 XML→CSV 변환을 일배치 형태로 안정 자동화, 수작업·임시 스크립트 운영 제거, 규모 증가 시 자동 확장으로 대응 |
| **상세 문서** | **[xml-image-csv-converter-자동화.md](./xml-image-csv-converter-자동화.md)** — 빌드업(수동 EC2→컨테이너·ECR→CI/CD→Airflow+EC2→AWS Batch), 워크플로우, Dockerfile 최적화 |
| **리드 평가** | “해야 할 일 한 것 / 기술 부채 스스로 정리” 수준으로 성과 1번만 정식 성과로 평가. |

## 6. 성과 리뷰·피드백 요약 (참고용)

### 6.1. 나 → 리드 (지원 요청·기대, 2025 상반기 성과 리뷰)

- **json-image-csv-converter** 개발 시 추출 metadata를 **DB화**해 AI researcher가 직접 쿼리할 수 있도록 설계하고 싶음.
- 단순 데이터 추출보다 **자동화 기반 데이터 파이프라인** 구성, **직접 기획·아이디어**를 내는 업무 희망.
- 아키텍처 설계 등 고민 시 형근님 도움 감사.

### 6.2. 하반기 기대치 (함께 논의)

- 검탐 스쿼드와 밀접 협업, **데이터 추출 + 모델 성능 개선** 방향 함께 고민.
- **GPU 서버 관리·모니터링**.

### 6.3. 리드 → 나 (하반기 성과리뷰)

- **강점**: 소통(Why에서 출발해 프로세스/프로젝트 맥락 파악·How 도출). “PM 하셔도 잘하실 것 같다”는 피드백.
- **보완**: 임팩트·높은 기준(타협 성향, 80점에서 멈추지 말고 90점 또는 다량 처리). 납기 미준수·오너십 관련 동료 피드백 있음. “전체를 위한 생각” 보강(동일 맥락 작업을 다른 시스템에 적용하는 등 주도적 확장).
- **다음 반기**: **목적조직 업무 위주**로 성과. Mordor 스쿼드 업무는 “AI Agent 엔지니어링”에 가깝이, **진정한 AIOps(AI 도구를 활용한 업무/사무 자동화)** 로 강조.

### 6.4. Growth (성장 회고·리드 코멘트)

- **셀프**: 문제 배경·구조 정의, 자동화·운영 관점 설계 주도. 6개월 후엔 팀·시스템 효율, 능동적 문제 정의, 일정·리스크 관리 강화 기대.
- **리드**: 소통 강점 유지. **기술적 사안에 타협 금지**(80점→90점 또는 75~80점 다량). **AI Agent 기반** 방향으로 역량 강화 권유(프론트/백 구분 의미 약화, 현재 스쿼드 업무 방향이 미래에 유리).
