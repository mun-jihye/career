| 구분 | 내용 |
|------|------|
| **성과** | 레이아웃 자동 생성 모델 연구에 필요한 **학습용 데이터 생성기 (xml2image-csv converter)** 개발 및 전달 |
| **기여도 / 내 역할** | • 요구사항에 맞는 **데이터 생성 파이프라인 설계·개발** (XML → Image, CSV 변환) <br>• **모듈화**하여 연구원·유관 부서가 쉽게 이용할 수 있도록 구현 <br>• Save Process(xml/idx → image·csv), Visualize Process(브라우저 미리보기), Element Image 추출 설계·구현 <br>• CSV 포맷(template metadata)·SIMPLE_TEXT 등 XML 구조 반영 문서화, 배포·브랜치 이력 정리 |

## 3. 성과리뷰에서의 피드백 요약 (참고용)

- **리드 → 나 (강점)**  
  XML2Image-Csv Converter 모듈화·고객(연구원) 편의성 구현, DinoV2 기반 inference·프로젝트 코드 구현.
- **리드 → 나 (개선·시도해볼 점)**  
  업무 기한 준수 연습(예: 3일 예상이면 3일 안에 보고).  
  AIOps(pre-processing, training log, model/data versioning, deploying) 전반 사이클 경험·성장, 연구 스쿼드에서 AI 연구원들이 필요로 하는 Engineering 문제 적극 해결 기대.
- **나 → 리드 (기대·강점)**  
  명확한 비전·방향 제시, 지속적 피드백·인정. 스쿼드 분리 후에도 관심·지원, DinoV2 개발 시 GPU 설정 등 적극적 지원에 대한 만족.

## 5. 사용 기술·키워드 일람 (이력서/직무 매핑용)

- **데이터·파이프라인**: XML, CSV, Image 변환, 학습용 데이터 생성기, 데이터 전처리
- **모델·평가**: DinoV2, 이미지 유사도, 정합성 평가, inference
- **개발 방식**: 모듈화, 연구원/내부 고객 편의성, 파이프라인 설계
- **언급된 확장 영역**: 요소 검색 API, 크롤링(Canva), prompt engineering, AIOps(전처리·학습·버저닝·서빙)

## 6. 이력서/이직 준비 시 활용 팁

- **Agent workflow·기획자 직무**와 직접 연결하려면:  
  "학습 데이터 생성 **워크플로우**(XML→Image·CSV) 설계·구현", "연구원이 쓰기 쉬운 **도구/파이프라인** 기획·개발"로 강조하면 좋음.
- **수치**가 있다면 보강 권장: 처리 건수, 지원한 연구/실험 횟수, 모듈 사용 부서 수 등 (Confluence 성과 문서나 위키에 있으면 추가).
- 2024만 보면 **데이터 생성기·평가 코드**가 중심이므로, 2025~2026의 RLSC·에이전트·Linter·n8n 등은 `docs/레브잇_이력서.md`와 합쳐서 "지속적으로 데이터·파이프라인·품질·자동화를 다뤄왔다"는 스토리로 묶을 수 있음.
