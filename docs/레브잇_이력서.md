## 3. 코드베이스/문서 기반 구체 경험

### 3.1. LLM·에이전트 기반 구조 추출 (핵심 매칭)

- **Agentic RLSC Structuriser (v1)**  
  - 4개 에이전트(Generator, Context, Evaluator, ProcessIterator) 협업으로 **절대 좌표(ALSC) → 상대 좌표(RLSC)** 변환.  
  - 반복 개선(Iterative Refinement)으로 구조 품질 향상.  
  → **“LLM 기반 서비스 설계·파이프라인”**, **“복잡한 추론을 에이전트 협업으로 분해”**로 서술 가능.

- **Tool-Calling RLSC Structuriser (v2 설계·구현)**  
  - **Single-Inference + Policy Injection**: Layout / Role / Label / SmartBlock 등 정책을 **Adapter 단위**로 주입, 한 번의 LLM 호출로 RLSC 추론.  
  - `applyLayout`, `assignRole`, `assignLabel`, `checkRLSC`, `finish` 등 **도구 호출**로 구조 생성·검수.  
  - 정책은 `specifications.md` 등으로 분리되어 **정의 변경 시 해당 Policy만 교체** 가능.  
  → **“Prompt engineering + 도구 호출 기반 제어”**, **“정책과 추론 로직 분리로 유지보수성·실험 용이성 확보”**로 서술 가능.

- **정량 데이터 기반 정책 재설계**  
  - 329,649건 RLSC ambiguity 분석(역할·레이아웃 혼동)을 바탕으로 **Role/Label 정책 확장·재정의**.  
  - “프롬프트만 수정해서는 해결되지 않는 문제”를 **정의 확장 + Policy 단위 업데이트**로 해소.  
  → **“데이터 기반 LLM 정책 개선”**, **“대규모 검수 데이터로 문제 정의·우선순위 설정”**으로 서술 가능.

### 3.2. 데이터 파이프라인·카탈로그 성격의 시스템

- **RLSC 추출 파이프라인**  
  - RLSC 추출 → Design Object 추출 → Text Capacity·썸네일 후처리 → DB 저장.  
  - pg-boss 큐(일반/배치)·워커 수 조정, `rlsc_results`·`design_objects`·`layouts` 연동.  
  → **“대량 레이아웃/구조 데이터의 추출·정규화 파이프라인”** — 상품 속성 추출·체계화와 유사한 “구조화 파이프라인” 경험으로 매핑 가능.

- **Design Object·컴포넌트 메타**  
  - RLSC에서 재사용 단위(Design Object) 추출, Content Signature·메타 저장.  
  → **“구조화된 데이터에서 재사용 가능한 단위 추출·카탈로그화”**로 정리 가능.

### 3.3. 품질·지표·규칙 검증 (비즈니스 영향)

- **RLSC Linter**  
  - 73개 이상 규칙으로 RLSC 품질 검수, **accept rate** 등 지표로 다운스트림 품질 관리.  
  - v2에서 **blocking / advisory** tier 분리로 “신뢰도 높은 신호”만 배포 게이트에 반영.  
  → **“LLM 출력 품질을 규칙·지표로 관리”**, **“False Positive/False Negative 트레이드오프 고려”**.

- **린터 규칙 검증 프로세스**  
  - 검수 결과 CSV 추출 → 영향도 분석 → **파라미터 sweep** → 오버피팅 검증 → 규칙 적용.  
  - Production DB 기준 Reject Category 분류·accept rate 시뮬레이션.  
  → **“데이터 기반 규칙/파라미터 최적화”**, **“정확도·운영 비용 균형”**.

### 3.4. 협업·문서화·실행

- **RLSC v2 킥오프·설계 문서**  
  - “왜 v2가 필요한가”, “Policy-Driven 원칙”, “실험 설계”를 Confluence·Figma·Jira(MOR-1141)와 연결해 공유.  
  → **“기술 방향을 비개발자 포함 이해관계자와 맞추는 문서화”**.

- **템플릿 처리 파이프라인·연동**  
  - 미캔 BE ↔ MORDOR 연동 기준 신규 템플릿 처리 단계 정리.  
  → **“외부 시스템과의 데이터/워크플로우 연동 설계”**.

- **차세대 카탈로그·구조화 시스템**
